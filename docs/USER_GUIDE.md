# Unflatten Studio 使用指南

本文档面向最终用户。读完之后主人应该能：

- 在 60 秒内导入图片、套相机、调参数、出图
- 看懂试拍表，按需对比 24 台虚拟相机
- 复制 `.ucamera` 配方存档、分享、复用
- 修改 Seed 让效果稳定复现

## 0. 安装与启动

参考 [`QUICKSTART.md`](./QUICKSTART.md) 把项目克隆到本地。

```bash
cd unflatten
./tool/flutterw run -d macos    # 桌面
./tool/flutterw run -d ios      # iPhone / iPad 模拟器
./tool/flutterw run -d android  # Android 模拟器 / 真机
```

应用启动后默认进入 **Camera Lab** 主工作区。

## 1. Camera Lab 主界面

打开后看到的画面：

- **顶部头**：当前图片名（如果有）+ 「打开图像」+ 「复制配方」按钮
- **中间大画布**：当前相机的实时预览
- **右侧 Camera DNA**：8 个调校滑块（仅桌面端）
- **底部配方条**：当前相机包内的 6 台配方横向滚动
- **左导航**：4 个相机包（仅桌面端）
- **底部按钮**：试拍表 + 调校 + 图像 + 复制（移动端）

桌面端三栏布局的断点是 `1100px`——窗口拉宽到 1100 以上自动从单列变三栏。

## 2. 60 秒上手（最小流程）

**步骤 1 · 导入图片**

点顶部「打开图像」（或移动端「文件夹」图标），从本地选一张 JPG / PNG / WebP / TIFF / HEIC。

图片会立刻出现在中间画布。如果不导入图片，应用会用内置 demo 场景（彩色测试卡）展示相机效果。

**步骤 2 · 选一台相机**

桌面端点左导航的 4 个相机包之一：Analog / Y2K / Optical / Mobile。每个包下方列出 6 台配方。

移动端在顶部有相机包 chip 切换，下方是配方条。横向滑动配方条选相机。

选中的相机会立即套到大画布上。配方条当前选中的格子有边框高亮。

**步骤 3 · 调参数**

桌面端在右栏 **CAMERA DNA** 调，移动端点「调校」按钮弹底部抽屉。

8 个滑块：

| 滑块 | 范围 | 作用 |
|---|---|---|
| 曝光 | -1.5 ~ 1.5 | 整体亮度偏移 |
| 对比 | -1 ~ 1 | 拉大或压扁明暗差距 |
| 饱和 | -1 ~ 1 | 颜色浓淡 |
| 冷暖 | -1 ~ 1 | 偏蓝 / 偏黄 |
| 颗粒 | 0 ~ 1 | 胶片颗粒强度（lite 模式下被跳过） |
| 暗角 | 0 ~ 1 | 四角变暗 |
| 光晕 | 0 ~ 1 | 高光溢出 |
| 闪光 | 0 ~ 1 | 正面直闪模拟 |

调节时画布实时更新。如果画布响应慢，先在试拍表里关「全实时」开关。

**步骤 4 · 复制配方**

点「复制配方」按钮，剪贴板得到一段 JSON：

```json
{
  "schema": "unflatten-camera/v1",
  "id": "warm-35",
  "name": "Warm 35",
  "pack": "analog",
  "seed": 3501,
  "body": {...},
  "lens": {...},
  "medium": {...},
  "capture": {...},
  "condition": {...},
  "protect": []
}
```

把这段 JSON 粘到任何编辑器、IM、笔记里存档。需要复刻效果时再粘回来（v0.2 支持导入，v0.1 暂未启用导入 UI）。

## 3. 24 台内置配方速查

### Analog（胶片与冲洗）

| ID | 名称 | 风格关键词 |
|---|---|---|
| `warm-35` | Warm 35 | 暖色负片、柔和高光、自然颗粒 |
| `cool-slide` | Cool Slide | 冷色反转片、深蓝阴影、高对比 |
| `silver-mono` | Silver Mono | 黑白银盐、亮部细腻 |
| `expired-summer` | Expired Summer | 过期胶片、褪色暖调、随机漏光 |
| `cross-shift` | Cross Shift | 交叉冲洗、高饱和、色相偏移 |
| `bleach-city` | Bleach City | 银漂、低饱和、硬对比 |

### Y2K Digicam（千禧数码）

6 台 Y2K 数码机效果模拟，包含正面直闪、CCD 噪点、色彩溢出。

### Optical（光学实验）

6 台光学缺陷模拟配方，包括径向畸变、色差、暗角、光晕等。

### Mobile Eras（手机年代）

6 台手机摄影配方，覆盖早期安卓、iPhone 4S / 5C 时代风格。

完整字段含义见 [`RECIPE_FORMAT.md`](./RECIPE_FORMAT.md)。

## 4. Camera DNA 五模块（怎么读配方 JSON）

`body / lens / medium / capture / condition` 五段，每段代表一个独立的物理层：

| 模块 | 物理含义 | 调整什么 |
|---|---|---|
| `body` | 传感器与基础噪声 | 动态范围、高光软度、基础噪声、饱和偏移 |
| `lens` | 镜头光学 | 焦段、畸变、边缘柔化、色差、暗角、Bloom、光晕 |
| `medium` | 成像介质（胶片 / 数字） | 颗粒、彩色噪点、对比、饱和、冷暖、暗部偏色、高光偏色 |
| `capture` | 拍摄时设置 | 曝光、白平衡、闪光强度、闪光衰减、欠曝、时间戳 |
| `condition` | 介质缺陷 | 灰尘、划痕、漏光、死像素、JPEG 压缩、磨损 |

调校面板上的 8 个滑块实际是这五模块参数的「快速入口」，不是 1:1 对应——比如「颗粒」取自 `medium.grain` 与 `body.baseNoise` 的最大值。

## 5. Contact Sheet 试拍表

点「试拍表」按钮（或移动端底部第一个按钮），弹出对话框显示**当前图片 × 24 台相机**的全部结果：

- 默认 6 列 × 4 行 = 24 格（窄屏自动 4 列 / 2 列）
- 每格下方有相机名 + 相机包标签
- 点击任意一格会自动选那台相机并关闭对话框

**性能开关**：对话框顶部有「全实时」Switch。

- 开启（默认）：24 格全部跑完整程序化效果，画质测试时帧率低
- 关闭：24 格只跑色彩矩阵 + 暗角 + Bloom + 闪光，跳过 4 种随机重操作

实测首帧渲染从 ~190ms 降到 ~140ms（widget test 数据）。

## 6. Seed 与缺陷签名

每台相机的 `seed` 决定它的随机效果布局——颗粒、灰尘、死像素、漏光的位置。

- **同 seed + 同 id**：在不同设备（iOS / Android / macOS / Windows / Linux）**像素级一致**
- **同 id 不同 seed**：视觉风格相似，细节布局不同
- **不同 id 不同 seed**：完全不同的相机

通过「复制配方」导出的 JSON 里 `seed` 字段固定，所以复刻效果时只需：

1. 选同一台相机（id 一致）
2. Seed 默认从配方读，不改

如果主人想微调细节但保持风格，改 seed；想换风格，换 id。

## 7. 移动端 vs 桌面端

### 桌面端（窗口 ≥ 1100px）

- 三栏布局：左导航 / 中画布 / 右 Camera DNA
- 调校直接在右侧实时调
- 适合精细控制 + 批量比较

### 移动端（窗口 < 1100px）

- 单列布局：顶部头 / 画布 / 配方条 / 底部按钮
- 调校通过底部「调校」按钮弹抽屉
- 适合快速选相机 + 套效果

### 跨端一致

任何相机配方 + seed 在两端渲染**像素级一致**——同一张图套同一台相机，iPhone 看到的颗粒分布和 Mac 完全相同（前提是输入图片一样）。

## 8. 不在 v0.1 范围

明确不做的功能（v0.2+ 才会出现）：

- **视频滤镜**——只处理静态图像
- **PSD / 图层编辑**——输入是单张图像
- **真实相机或胶片品牌**——所有配方都是化名
- **云端账号 / 同步**——本地优先，素材默认不上传
- **导入 `.ucamera` 配方**——v0.1 只能复制导出，导入 UI 在 v0.2
- **Structure Engine**（主体拆分、背景补全等）——Camera Lab 之外的引擎，v0.2 起步

## 9. 故障排查

| 现象 | 原因 | 解决 |
|---|---|---|
| 导入图片没反应 | 文件超过 50MB 或格式不识别 | 转 JPG/PNG，控制大小 |
| 试拍表打开很慢 | 24 格全实时 | 关掉「全实时」开关 |
| 调节滑块没视觉变化 | 配方参数边界 | 该参数已到 min/max，尝试其他滑块 |
| 复制配方按钮点了没反应 | 剪贴板权限未授权（macOS 14+） | 系统设置 → 隐私 → 剪贴板 |
| 时间戳一直显示 | 配方 `capture.timestamp = true` | 当前版本无法 UI 关闭，v0.2 加 |
| 应用启动后是 demo 场景 | 没导入图片 | 点「打开图像」导入本地图片 |
| `flutter pub get` 超时 | 网络问题 | 加 `PUB_HOSTED_URL=https://pub.flutter-io.cn` 重试 |
